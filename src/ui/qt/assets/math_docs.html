Detección del Foco — Mínimo de la Curva Métrica

⟵ src/analysis/focus/FocusDetector.cpp

El foco se define como la posición axial s* donde la métrica M(s)

alcanza su mínimo global:

s* = argmin_s { M(s) } con M(s) = r_rms(s) [por defecto]

Confianza del foco (contrast proxy)

Se mide el contraste alrededor del bin óptimo i*:

C = [ M(i*−1) + M(i*+1) ] / 2 − M(i*) ℹ C > 0 indica un mínimo real (pozo); C ≈ 0 indica curva plana (sin foco claro). ℹ Para datos Geant4 de muones cósmicos (sin lente) el foco es el punto de ℹ menor dispersión transversal, que puede estar en la entrada del volumen.

Métricas disponibles

r_rms = √(σ_u² + σ_v²) [radio RMS total — por defecto] σ_u [desviación estándar en la dirección transversal u] σ_v [desviación estándar en la dirección transversal v]

Estadísticas por Frame — Radio RMS Transversal

⟵ src/analysis/statistics/FrameStatisticsEngine.cpp

Para cada bin axial se acumulan los puntos de todas las trayectorias

y se calculan las siguientes estadísticas transversales:

Proyección al marco de referencia

Sea p⃗ la posición de un muestra y ê_s, ê_u, ê_v los versores del marco:

s = (p⃗ − origen) · ê_s [coordenada longitudinal] u = (p⃗ − origen) · ê_u [transversal 1] v = (p⃗ − origen) · ê_v [transversal 2]

Momentos estadísticos (acumulador de un solo paso)

Con N puntos en el bin:

ū = (1/N) Σᵢ uᵢ [centroide u] v̄ = (1/N) Σᵢ vᵢ [centroide v] σ_u² = (1/N) Σᵢ uᵢ² − ū² [varianza u] σ_v² = (1/N) Σᵢ vᵢ² − v̄² [varianza v]

Radio RMS transversal (proxy de foco)

r_rms = √(σ_u² + σ_v²) ℹ El foco geométrico se define como el bin con r_rms mínimo. ℹ Se usa el estimador sesgado (1/N) por consistencia con el proxy de foco; ℹ para N >> 1 la diferencia con el estimador insesgado 1/(N-1) es despreciable.

Modos de binning

Uniforme: bins de igual anchura Δs = (s_max − s_min) / N_bins

Equal-count: bins con igual número de puntos (percentiles).

Sincronizado: usa el índice de muestra como "tiempo" (requiere ≥80 % de

trayectorias con el mismo número de pasos).

Detección de Planos de Scoring — Análisis de Densidad

⟵ src/biosim/core/ScoringPlaneDetector.cpp

Los planos de scoring (entrada, salida, contador) se detectan automáticamente

analizando la densidad de puntos a lo largo del eje longitudinal.

Histograma de densidad axial

Se divide el rango axial [s_min, s_max] en N_bins bins de ancho uniforme:

Δs = (s_max − s_min) / N_bins

Cada bin acumula los puntos de todas las trayectorias cuya coordenada s cae

en su intervalo [s_min + k·Δs, s_min + (k+1)·Δs).

Detección de flancos (transiciones)

Sea P_peak el conteo máximo del histograma. Se definen dos umbrales:

umbral_alto = f_high · P_peak (por defecto f_high = 0.30) umbral_bajo = f_low · P_peak (por defecto f_low = 0.05)

Flanco ascendente (entrada): bin k con P[k] &lt; umbral_bajo → P[k+1] > umbral_alto

Flanco descendente (salida): bin k con P[k] > umbral_alto → P[k+1] &lt; umbral_bajo

ℹ Sólo se marcan flancos con al menos min_crossings trayectorias (por defecto 10).

Plano contador genérico

Se añade automáticamente un plano contador en la posición media del haz:

s_mid = (s_min + s_max) / 2

Resolución del Marco de Referencia del Haz

⟵ src/io/normalization/AxisFrameResolver.cpp

El marco de referencia (axis frame) transforma coordenadas de laboratorio

(x, y, z) a coordenadas del haz (s, u, v), donde s es el eje longitudinal.

Detección automática del eje longitudinal

Se calcula el rango de cada coordenada sobre todos los puntos del dataset:

Rₓ = max(x) − min(x), R_y = max(y) − min(y), R_z = max(z) − min(z)

El eje con mayor rango es el eje longitudinal del haz:

ê_s = ê_z si R_z ≥ R_x y R_z ≥ R_y (caso más común en Geant4)

Signo del eje longitudinal

Se suma el vector (último_punto − primer_punto) de cada trayectoria:

Σ_z = Σᵢ ( z_final_i − z_inicial_i ) signo(ê_s) = +1 si Σ_z ≥ 0, −1 si Σ_z &lt; 0 ℹ Esto garantiza que ê_s apunta en la dirección de propagación del haz.

Marco resultante

Para eje Z positivo (caso del CSV de muones):

ê_s = (0, 0, +1), ê_u = (1, 0, 0), ê_v = (0, 1, 0)

Confianza del marco

confidence = R_max / (Rₓ + R_y + R_z) ∈ [1/3, 1] ℹ confidence → 1 cuando el haz es puramente axial; → 1/3 cuando las tres ℹ extensiones son iguales (ambigüedad).

Normalización del Dataset — Resúmenes por Trayectoria

⟵ src/io/normalization/DatasetNormalizer.cpp

La normalización ordena las muestras de cada trayectoria por tiempo y

calcula los resúmenes físicos.

Ordenación temporal

Las muestras de cada trayectoria se ordenan por t [s] ascendente.

Longitud de trayectoria (range)

Se calcula como la suma de las distancias euclídeas entre pasos consecutivos:

L = Σᵢ |p⃗ᵢ − p⃗ᵢ₋₁| con |Δp⃗| = √(Δx² + Δy² + Δz²)

Energía total depositada

E_dep_total [MeV] = Σᵢ edep_MeVᵢ E_dep_total [eV] = E_dep_total [MeV] · 10⁶

Energías de entrada y salida

E_entry = kinE_MeV del primer paso (después de ordenar por t) E_exit = kinE_MeV del último paso ℹ E_entry ≈ E_gun para el primer paso del primario en Geant4.
